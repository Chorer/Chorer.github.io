「译」Web 开发新手应该知晓的20件事

 2019-09-21   
 1.1k words    3 mins   
  • 原文地址：20 Things That Are Good to Know When You Start Web Developing
  • 原文作者：Daan
  • 译者：Chor

在我当初刚从事 web 开发的时候，有很多重要的事我并没有事先了解。现在看来，我的很多期望都和现实有很大的差距。在这篇文章里，我会告诉你20 件事情，这些都是在你准备开始或者刚开始 web 开发不久的时候就应该知晓的。这有助于你更好地正视自己的期望，同时少走一些弯路。

1.编程和学位无关

编程可以随时开始，并不需要学位。在互联网上你可以找到很多资源，尤其是一些基础知识。你完全可以自学编程，同时求助于互联网。

2.Googling 是一项技能

因为你刚开始学习 web 开发，所以并没有足够的知识可以解决当前遇到的问题。这是正常的，但并不意味着你不能解决这些问题。知道怎么 google 是一项重要的技能，可以帮助你节省大量时间。

3.你不能什么都学

有太多东西要学了。单看 JavaScript 的流行框架，就有：React，Vue 和 Angular。你没办法学习所有的框架，并且也没有这个必要。你只需要专注于你喜欢的或者公司目前使用的框架即可。

4.最难的事是写简单的代码

很多没有经验的开发者会写一些花里胡哨的代码，以此向其他开发者炫耀自己的编码能力。请务必不要这么做，你应该尽可能地保持代码简洁。

5.没有时间做全面测试

根据我的经验，开发者对测试工作比较懈怠。大部分的开发者都觉得测试环节不是很有趣。最后，你是一个制造东西而不是破坏东西的开发者。
我们是有最后期限的，大部分时候你都需要处理它。一旦意识到有可能错过最后期限，第一件应该放弃的事就是测试。尽管大家都知道这不利于最终结果，但的确是这样做的，

6.

7.当回顾旧代码的时候，你会觉得很丢脸

当你刚开始从事开发的时候，你想做的只是修复一些东西。只要代码可以运行就够了。对于那些缺乏经验的开发者来说，可以运行的软件和运行良好的软件没有差别。但随着你成长为一名真正意义上的开发者，你将不得不面对以前写的旧代码，然后心想：“这些面条式代码真的是我之前写的吗？”好吧，在这种情况下你只需要做一件事：好好嘲笑一番，然后重构一下这些乱糟糟的代码。

8.你将花费大量时间查看 bugs

调试代码是你的一个主要工作。写出完全正确的代码是不可能的，尤其是在你经验尚缺的时候。缺乏经验的开发者之所以需要花大量时间去调试，是因为他们不知道应该查看哪里，有时候他们甚至不知道应该找什么。最糟糕的是，大部分时候，这些 bugs 是你自己造成的。

9.IE 是最差的浏览器

Internet Explorer，又名 Internet Exploder，将会让你对自己书写 的 CSS 感到厌恶。即使是最基本的样式，在 IE 浏览器中也会崩掉。在某个时候，你将会疑惑为什么会存在这么多不同的浏览器。对于这个问题，大多数公司采取的做法是只支持 IE11 或者更新的版本。

10.

11.你会假装自己理解了同事所说的东西

你的同事可能会和你聊聊最新的技术或者工具，你在这个过程中会对他的看法频频点头。但事实是你并不理解他说的那些东西。

12，你不需要记住所有的东西

编程就是知识的应用，没有必要记住所有的东西。你随时可以在网上找到资源，只需要知道去哪里找就可以了。做项目、积累经验，这些东西自然就记住了。
